Critical #1
#1 Race Condition — insertNextReading readingCount 충돌 시 메시지 노출
ReviewService.java
문제 findMaxReadingCount() 조회 후 save()까지 락이 없어 동시 요청 시 같은 readingCount로 충돌 가능. DB 유니크 제약이 잡아도 Hibernate 내부 오류 메시지가 그대로 클라이언트에 노출됨.
수정 reviewRepository.save()를 try-catch로 감싸, DataIntegrityViolationException 발생 시 사용자 친화적인 메시지로 변환하여 재던짐. DB 제약이 최종 방어선으로 작동하되 메시지는 서비스 레이어에서 제어.

Critical #3
#3 AccessDeniedException — Spring Security 필터 레벨로 위임
SecurityConfig.java GlobalExceptionHandler.java
문제 @RestControllerAdvice가 AccessDeniedException을 가로채면 Spring Security의 ExceptionTranslationFilter가 동작하지 않아 미인증(401)과 인가 실패(403)를 구분하지 못함. OAuth2 세션/리다이렉트 흐름도 끊길 수 있음.
수정 SecurityConfig에 .exceptionHandling() 추가. 미인증은 authenticationEntryPoint(401 JSON), 인가 실패는 accessDeniedHandler(403 JSON)로 각각 처리. GlobalExceptionHandler의 AccessDeniedException 핸들러는 제거.
📁 config/SecurityConfig.java · securityFilterChain()
.sessionManagement(...)
+ .exceptionHandling(ex -> ex
+ .authenticationEntryPoint((req, res, e) -> {
+ res.setStatus(401);
+ res.getWriter().write("{\"message\": \"로그인이 필요합니다.\"}");
+ })
+ .accessDeniedHandler((req, res, e) -> {
+ res.setStatus(403);
+ res.getWriter().write("{\"message\": \"접근 권한이 없습니다.\"}");
+ })
+ )
.logout(...)
