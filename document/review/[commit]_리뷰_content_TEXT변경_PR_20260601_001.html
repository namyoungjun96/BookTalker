[commit] PR — Review content 컬럼 타입 TEXT 변경

📅 2026-06-01 🌿 feat/mypage-content-preview → main 🗂 review / backend

변경 요약

Review 엔티티의 content 필드 타입을 JPA 기본값인 varchar(255)에서 TEXT로 변경합니다. 독후감 본문의 글자 수 제한을 제거하여 사용자가 자유롭게 기록할 수 있도록 합니다.

변경 이유

varchar(255)의 한계: 한글 기준 약 85자로, 독후감을 자유롭게 기록하는 서비스 특성과 맞지 않음.
글자 수 제한의 부적절성: 독후감은 개인 기록물 성격이 강하므로 제한을 두는 것이 UX상 바람직하지 않음.
PostgreSQL TEXT: 저장 가능한 최대 크기 제한 없음 (실질적으로 1GB).

코드 변경

// Review.java

- private String content;
+ @Column(columnDefinition = "TEXT")
+ private String content;

변경 파일

구분파일변경 내용
BE review/entity/Review.java content 필드에 @Column(columnDefinition = "TEXT") 추가 (1줄)

⚠️ 운영 DB 별도 조치 필요

JPA 엔티티 수정만으로는 기존 운영 DB 컬럼 타입이 변경되지 않습니다.
배포 전 운영 DB에 아래 쿼리를 직접 실행해야 합니다.

ALTER TABLE review ALTER COLUMN content TYPE TEXT;

기존 데이터가 255자 이하라면 데이터 손실 없이 즉시 적용 가능합니다.

테스트 체크리스트

  • 255자 초과 독후감 작성 후 저장 성공 확인
  • 저장된 긴 본문이 상세 페이지에서 정상 표시되는지 확인
  • 운영 DB ALTER 쿼리 실행 후 기존 데이터 정상 조회 확인
