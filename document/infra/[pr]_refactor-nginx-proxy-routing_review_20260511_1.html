Summary

  • 백엔드 포트(8010) 직접 호출 방식에서 nginx 리버스 프록시 방식으로 전환하면서 필요한 설정을 정리합니다.
  • OAuth2 콜백 redirect-uri를 nginx 경유 경로(/api/login/oauth2/code/naver)로 수정합니다.
  • 프론트 환경 변수를 local / dev / staging / prod 4단계로 명확히 분리합니다.
  • 각 Vue 컴포넌트에 하드코딩된 /api prefix를 일괄 제거합니다.

Changes

  • [Backend] application-prod.yml — OAuth2 redirect-uri: :8010/login/oauth2/code/naver → /api/login/oauth2/code/naver
  • [Frontend] .env.development — local API: http://localhost:8010
  • [Frontend] .env.prod — prod API: http://161.118.143.129/api
  • [Frontend] .env.dev / .env.staging — 미정(빈값)으로 정리
  • [Frontend] package.json — build:staging을 --mode staging으로 수정, build:dev 추가
  • [Frontend] 9개 컴포넌트에서 /api/book, /api/review, /api/auth 등 하드코딩된 prefix 일괄 제거

의사결정

  • 왜 /api prefix를 env에만 두는가 — 컴포넌트가 baseURL을 모르게 하기 위함. local은 http://localhost:8010, prod는 http://161.118.143.129/api가 baseURL에 포함되므로 컴포넌트 경로에 /api를 쓰면 prod에서 이중 경로가 됨.
  • SecurityConfig redirectionEndpoint 미수정 — nginx 설정으로 /api/login/oauth2/code/naver를 백엔드 콜백 경로로 포워딩하도록 처리함.

Test Plan

  • prod 빌드(npm run build:prod) 후 nginx를 통한 API 호출 정상 확인

  • build:staging 실행 시 .env.staging 파일이 로드되는 것 확인
