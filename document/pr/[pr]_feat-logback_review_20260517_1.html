PR 본문 초안

FEAT: logback 프로파일 분리 및 Discord ERROR 알림 추가

Summary

  • logback을 local / prod 프로파일로 분리하여 환경별 로그 출력 전략을 명확히 합니다.
  • prod 환경에서 ERROR 로그 발생 시 Discord Webhook으로 실시간 알림을 전송합니다.
  • Java 표준 HttpClient 기반의 커스텀 DiscordWebhookAppender를 구현합니다.
  • app.discord.webhook-url을 환경 변수(DISCORD_WEBHOOK_URL)로 주입받아 민감 정보를 코드에서 분리합니다.

Changes

  • [신규] DiscordWebhookAppender.java — AppenderBase<ILoggingEvent> 상속, 연결 타임아웃 3s / 요청 타임아웃 5s, fail-safe 처리
  • [신규] additional-spring-configuration-metadata.json — app.discord.webhook-url 커스텀 프로퍼티 등록 (IDE 경고 억제)
  • [수정] logback-spring.xml — scan="true" scanPeriod="60s", springProfile 분리, ERROR_FILE / DISCORD appender 추가
  • [수정] application-prod.yml — app.discord.webhook-url: ${DISCORD_WEBHOOK_URL:} 추가

의사결정

  • 왜 외부 라이브러리 없이 Java HttpClient를 사용했는가 — logback appender는 Spring 컨텍스트 외부에서 초기화되므로 Spring Bean 주입이 불가. 별도 의존성 없이 JDK 11+ 표준 HttpClient로 구현.
  • 왜 webhookUrl을 빈값 허용(:)으로 설정했는가 — local 환경에서 환경 변수 미설정 시 appender가 조용히 skip하도록 fail-safe 처리. prod에서만 실제 URL 주입.
  • 왜 logback namespace를 logging.*가 아닌 app.*로 했는가 — logging.*는 Spring Boot 예약 네임스페이스. 커스텀 프로퍼티는 app.*로 분리해 IDE 경고 및 충돌 방지.

Test Plan

  • local 환경에서 앱 실행 시 콘솔에 DEBUG 로그 출력 확인
  • prod 환경에서 ./logs/booktalker.log, ./logs/booktalker-error.log 생성 확인
  • DISCORD_WEBHOOK_URL 환경 변수 설정 후 ERROR 로그 발생 시 Discord 채널 알림 수신 확인
  • DISCORD_WEBHOOK_URL 미설정(빈값) 시 예외 없이 정상 동작 확인
