[design] BookReviewsView 네비게이션 탭 활성화 버그 수정

📅 2026-06-01 🗂 mypage / frontend 🔖 설계 검수 요청

문제 정의

랭킹 페이지에서 책을 클릭해 /books/:isbn13/reviews (BookReviewsView)에 진입하면 상단 네비게이션의 어떤 탭도 활성화되지 않음. 동일 증상이 반복적으로 발생.

원인 분석

useNav.js의 isActiveNav는 아래 로직으로 활성 탭을 결정합니다. 

// useNav.js
const isActiveNav = (path) => {
  if (route.name === 'review-create') { ... }
  return (route.meta.activeNav ?? route.path) === path;
};

meta.activeNav가 없으면 실제 경로(/books/9788934975236/reviews)를 그대로 사용 → 네비게이션 탭 경로(/, /mypage, /book-search)와 일치하는 것이 없음. 

// router/index.js — book-reviews 라우트
{
  path: '/books/:isbn13/reviews',
  name: 'book-reviews',
  component: BookReviewsView,
  // meta.activeNav 누락!
}

참고로 review-detail, my-book-detail은 이미 meta: { activeNav: '/mypage' }가 있어 정상 동작 중.

해결 방안 비교 (3가지)

A. router/index.js에 meta.activeNav 추가 권장 난이도 ⭐
기존 패턴과 동일하게 라우트 정의에 meta: { activeNav: '/' } 한 줄 추가. 
// router/index.js
{
  path: '/books/:isbn13/reviews',
  name: 'book-reviews',
  component: BookReviewsView,
  meta: { activeNav: '/' },
}
장점단점
기존 패턴 일관성 유지
라우터 파일에서 한눈에 관리
useNav.js 수정 불필요		 없음
B. useNav.js에 경로 패턴 매칭 추가 난이도 ⭐⭐
isActiveNav 함수 내에서 /books/로 시작하는 경로면 홈(/) 활성화. 
// useNav.js
const isActiveNav = (path) => {
  if (route.path.startsWith('/books/')) return path === '/';
  ...
};
장점단점
라우터 파일 수정 불필요 경로 패턴이 늘어날수록 useNav.js가 복잡해짐
라우터와 네비게이션 로직이 분산됨
C. AppHeader.vue에서 route.name으로 분기 난이도 ⭐⭐
헤더 컴포넌트에서 직접 route.name === 'book-reviews'일 때 홈 탭 활성화 조건 추가.
장점단점
라우터 파일 수정 불필요 활성화 로직이 헤더 컴포넌트에 섞임
관심사 분리 위반
유지보수 어려움

권장 방안 및 근거

A안 채택 권장.
review-detail, my-book-detail 등 기존 라우트가 이미 meta.activeNav 패턴을 사용 중이므로 일관성을 유지하는 것이 가장 적합합니다. 변경 범위도 라우터 파일 한 줄로 최소화됩니다.
