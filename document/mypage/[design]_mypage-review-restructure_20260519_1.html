1 · 구조 변경 개요 (현재 vs 변경 후)

[ 현재 ] /mypage └── 모든 리뷰 flat 목록 (책 A 1회독 / 책 A 2회독 / 책 B 1회독 ...) 카드마다: 표지 + 제목 + 한줄요약 + 회독 뱃지 ───────────────────────────────────── ⭐ 4 [한번 더 봤어요] [🗑 삭제하기] [ 변경 후 ] /mypage ← 책 단위 목록 └── 책 카드 (책 A · 3회독까지 · 최근 날짜) 책 카드 (책 B · 1회독까지 · 최근 날짜) ... ↓ 클릭 /mypage/books/:isbn13 ← 해당 책의 회독 전체 목록 └── 1회독 | 2025-03-01 | ⭐4 | 한줄요약 ... | [삭제] 2회독 | 2025-08-10 | ⭐5 | 한줄요약 ... | [삭제] 3회독 | 2026-01-22 | ⭐4 | 한줄요약 ... | [삭제] [+ 한번 더 읽기] 버튼 (상단 고정)

2 · 구현 계획

항목 내용
수정 MyPageView.vue /review/list 응답을 bookIsbn13 기준으로 프론트에서 그룹핑.
각 그룹에서 최대 readingCount + 가장 최근 regDate를 대표값으로 사용.
카드 클릭 → /mypage/books/:isbn13 이동. "한번 더 읽기" 버튼은 이 화면에서 제거.
신규 MyBookDetailView.vue /review/list 재호출 후 isbn13로 필터링 (백엔드 변경 없음).
기본 정렬: readingCount 오름차순 (1→2→3회독).
정렬 토글: 회독순 ↕ / 최신순 ↕.
상단: 책 커버 + 제목 + [+ 한번 더 읽기] 버튼.
각 행: 회독 수 뱃지 + 날짜 + ⭐ 별점 + 한줄요약 + [삭제] 버튼.
수정 router/index.js { path: '/mypage/books/:isbn13', name: 'my-book-detail', meta: { activeNav: '/mypage' } } 추가
백엔드 변경 없음. /review/list 그대로 사용. 프론트에서 그룹핑/필터링 처리.

4 · UI 개선 (#3) — 제 의견

지금 카드 하단 액션바 문제 동의합니다
현재 카드는 책 정보 영역 아래에 별점 + 긴 버튼 2개가 가로 한 줄로 붙어 있어서, 모바일(375px)에서 버튼이 줄바꿈되거나 텍스트가 잘립니다. "한번 더 봤어요"라는 텍스트 자체도 너무 길어 버튼 크기를 낭비합니다.
구조 개편하면 자연스럽게 해소됩니다
MyPageView가 책 단위 카드로 바뀌면 더 이상 카드마다 "한번 더 읽기" 버튼이 필요 없습니다. "한번 더 읽기"는 MyBookDetailView 상단에 1개만 두면 됩니다. 삭제도 회독 행마다 소형 아이콘 버튼 하나로 충분합니다.
책 카드에서 별점 표기 — 선택 사항
책이 여러 회독이면 "어떤 회독의 별점"을 보여줄지 모호해집니다. 선택지:
A. 가장 최근 회독 별점만 표시 (심플)
B. 별점 표시 안 함, detail 페이지에서만 표시 (혼란 없음)

저는 B (표시 안 함)을 추천합니다. 카드에서는 "몇 회독까지 읽었는지"와 "최근 날짜"만 알면 충분합니다.
MyBookDetailView의 회독 행 UI 제안
회독 행을 세로 카드가 아니라 가로 compact row로 만들면 모바일에서도 깔끔합니다.

[1회독 뱃지] 날짜 ⭐ 4 [🗑]
"한 줄 요약 텍스트 (클릭하면 detail 이동)"

뱃지 + 날짜 + 별점 + 삭제를 한 줄에 두고, 그 아래 한줄요약을 배치하면 카드 높이를 반으로 줄일 수 있습니다.

→ 1, 2번 구현 방향이 확정되면 3번 UI는 MyBookDetailView 작성 시 함께 반영하겠습니다.

5 · 확인 요청

구현 진행 전 확인이 필요한 항목

① MyPageView 책 카드에서 별점 표시 여부 — A(최근 회독 별점) / B(표시 안 함)

② MyBookDetailView에서 회독 행 클릭 시 동작 — 기존 detail 페이지(ReviewDetailView)로 이동? 또는 인라인 펼침?

③ 삭제 버튼 위치 — 행마다 🗑 아이콘 버튼 / 또는 스와이프로 삭제(모바일 UX)?
