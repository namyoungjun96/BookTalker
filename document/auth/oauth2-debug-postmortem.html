PWA OAuth2 로그인 불가 디버깅

BookTalker · Spring Boot + Naver OAuth2 + Vite PWA
TL;DR

네이버 로그인 버튼 클릭 시 OAuth2 엔드포인트로 이동하지 않고 서비스 로그인 페이지로 튕기는 현상. nginx, Spring Boot 순으로 디버깅했으나 실제 원인은 Service Worker가 window.location.href 요청을 가로채 캐시된 index.html을 반환하고 있었던 것. navigateFallbackDenylist에 /api/ 경로를 추가하여 해결.

STEP 01

문제 현상

→
로그인 버튼 클릭
window.location.href = 'https://booktalker.co.kr/api/oauth2/authorization/naver'
→
서비스 자체 로그인 페이지로 리다이렉트 UNEXPECTED
네이버 로그인 페이지로 이동해야 정상
→
Spring Boot 로그에 /oauth2/authorization/naver 요청 없음 NOT REACHED
→
nginx 액세스 로그에도 해당 요청 없음 NOT REACHED
STEP 02

계층적 디버깅 과정

1

Spring Boot 설정 의심

SecurityConfig에서 oauth2Login() 설정, authorizeHttpRequests 설정 확인. formLogin과 충돌 여부 점검.

이상 없음
2

nginx 설정 의심

location /api/ 블록이 /oauth2/authorization/naver를 제대로 프록시하는지 확인. nginx -T로 전체 설정 점검.

이상 없음
3

curl로 Spring Boot 직접 테스트

nginx를 거치지 않고 직접 Spring Boot 포트로 요청. 302로 네이버 로그인 URL 생성 확인.

# nginx 없이 직접 테스트
curl -v http://localhost:8010/oauth2/authorization/naver

# 결과: 302 → nid.naver.com/oauth2.0/authorize?...
→ Spring Boot 정상 확인
Spring Boot 정상
4

nginx 액세스 로그 실시간 확인

브라우저에서 요청 시 nginx 로그에 /api/oauth2/authorization/naver 요청 자체가 없음을 확인. 서버 도달 전에 처리됨을 확정.

nginx 정상
5

브라우저 Network 탭 확인

/api/oauth2/authorization/naver 요청이 200으로 끝남 확인. Size 컬럼에서 (ServiceWorker) 표시 발견. Service Worker가 요청을 가로채 캐시된 index.html을 반환하고 있었음.

원인 발견!
STEP 03

근본 원인

→
Vite PWA의 Workbox가 기본적으로 모든 navigation 요청을 처리 ROOT CAUSE
navigateFallback 설정 시 Service Worker가 모든 페이지 이동을 가로채 index.html 반환
→
window.location.href도 navigation 요청으로 분류됨
/api/oauth2/authorization/naver도 예외 없이 Service Worker에게 가로채임
→
nginx, Spring Boot까지 요청이 도달하지 못함
STEP 04

해결

Workbox 설정에 navigateFallbackDenylist를 추가해 /api/ 경로는 Service Worker가 처리하지 않도록 제외.

JAVASCRIPT vite.config.js 
workbox: {
  // Before: navigateFallback 관련 설정 없음
  // → Workbox가 모든 navigation 요청을 index.html로 처리

  // After
  navigateFallback: 'index.html',
  navigateFallbackDenylist: [/^\/api\//],  // ← 추가

  runtimeCaching: [
    // 기존 설정 유지
  ]
}

⚠️ 배포 후 브라우저에서 개발자 도구 → Application → Service Workers → Unregister 후 테스트 필요. 이전 버전 Service Worker가 브라우저에 남아있을 수 있음.

LESSON

배운 것들

🔍

계층적 디버깅

브라우저 → nginx → Spring Boot 순서로 어느 레이어까지 요청이 도달하는지 확인하는 것이 핵심. curl로 nginx 우회, nginx 로그로 Spring Boot 도달 여부 확인.

⚡

Service Worker의 범위

PWA의 Service Worker는 window.location.href 같은 navigation 요청도 가로챌 수 있음. API 경로는 반드시 denylist에서 제외해야 함.

🔄

SSL Termination 패턴

nginx에서 HTTPS를 처리하고 내부는 HTTP로 통신하는 것이 정석. Spring Boot에 forward-headers-strategy: native 설정으로 프록시 인식.

🚫

로그에 없으면 미도달

Spring Boot 로그에 요청이 없으면 도달하지 않은 것. nginx 로그에도 없으면 nginx조차 못 간 것. 로그 부재 자체가 단서.

💼 면접 포인트

"PWA 개발 중 OAuth2 로그인이 동작하지 않는 문제를 디버깅한 경험이 있습니다. nginx 로그, Spring Boot 로그, 브라우저 Network 탭을 순서대로 확인하며 요청이 어느 레이어까지 도달하는지 추적했고, Service Worker가 window.location.href 요청을 가로채고 있음을 발견했습니다. Workbox의 navigateFallbackDenylist로 API 경로를 제외하여 해결했습니다."
