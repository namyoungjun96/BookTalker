수정 내용

#1 nextReadingCount가 항상 '?' 로 표시되는 문제

원인: nextReadingCount = ref('?')로 고정. 서버 호출 없이 표시용으로만 남겨진 상태.

해결: MyPageView에서 이미 보유 중인 review.readingCount를 query param으로 전달, ReviewCreateView에서 computed(() => Number(route.query.readingCount || 1) + 1)로 계산. 서버 추가 호출 없음.

표시 예시: 현재 2회독 리뷰 → query readingCount=2 → 화면 3회독 독후감 작성

#2 독후감 작성 화면에서 nav 활성화가 안 되는 문제

원인: review-create 라우트에 meta.activeNav가 없어서 isActiveNav가 /reviews/new와 비교 → 어떤 nav 항목도 매칭 안 됨.

해결: useNav.js의 isActiveNav에 review-create 분기 추가.

검색 → 1회독 독후감 작성: 검색 탭 활성화
마이페이지 → 회독 독후감 작성: 마이페이지 탭 활성화

데이터 흐름 (nextReadingCount)

[ 기존 ] MyPageView → router.push({ query: { mode, isbn13, title, cover } }) ReviewCreateView → nextReadingCount = ref('?') → '?회독 독후감 작성' [ 변경 후 ] MyPageView → router.push({ query: { mode, isbn13, title, cover, readingCount: review.readingCount } }) ReviewCreateView → nextReadingCount = computed(() => Number(route.query.readingCount || 1) + 1) → '3회독 독후감 작성' (예: readingCount=2 → 2+1=3)
