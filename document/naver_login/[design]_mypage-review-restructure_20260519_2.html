[설계 v2] 마이페이지 리뷰 구조 개편

v1 대비 변경: 단독 회독 사용자에게 불필요한 depth가 생기는 문제를 해결하기 위해 카드 클릭 동작을 회독 수에 따라 분기하는 방식으로 수정.

1 · 핵심 문제 (v1에서 제기된 이슈)

단독 회독 사용자에게 불필요한 depth
책을 한 번만 읽은 사람이 카드를 클릭하면 → MyBookDetailView로 이동
→ 그 안에 리뷰가 딱 1개만 있음
→ 의미 없는 중간 단계가 하나 추가된 것처럼 느껴짐
해결: 클릭 동작을 회독 수에 따라 분기
이미 MyPageView에서 isbn13 기준으로 그룹핑을 하므로, 그룹의 리뷰 개수가 1개면 바로 ReviewDetailView, 2개 이상이면 MyBookDetailView로 보냄.
추가 API 호출 없이 조건문 하나로 해결.

[ MyPageView — 책 단위 카드 ] 책 A (1회독) ──────────────────────────────→ ReviewDetailView (바로 이동) 책 B (3회독) ──→ MyBookDetailView ──→ 회독 1 선택 ──→ ReviewDetailView 책 C (2회독) ──→ MyBookDetailView ──→ 회독 2 선택 ──→ ReviewDetailView [ 카드에 표시할 정보 ] 모든 책: 표지 + 제목 + 최근 날짜 다독 책: "3회독까지" 뱃지 추가 단독 책: 뱃지 없음 (또는 "1회독" — 선택 사항)

5 · 구현 계획

항목변경 내용
수정 MyPageView.vue /review/list 응답을 bookIsbn13 기준으로 그룹핑.
카드 클릭: group.reviews.length === 1 → review-detail (id 전달),
            group.reviews.length >= 2 → my-book-detail (isbn13 전달).
카드 대표값: 최대 readingCount + 가장 최근 regDate.
"한번 더 봤어요" 버튼 제거 (MyBookDetailView 상단으로 이동).
신규 MyBookDetailView.vue /review/list 재호출 후 isbn13로 필터링.
정렬: readingCount 내림차순 (3→2→1, 최신 회독이 먼저).
상단: 책 커버 + 제목 + [+ 한번 더 읽기] 버튼.
각 행: 회독 뱃지 + 날짜 + ⭐ 별점 + 한줄요약 + [🗑] 버튼.
행 클릭 → ReviewDetailView.
수정 router/index.js { path: '/mypage/books/:isbn13', name: 'my-book-detail', meta: { activeNav: '/mypage' } } 추가.
백엔드 변경 없음. /review/list 그대로 사용.

7 · 남은 확인 사항

① 회독 행 클릭 — ReviewDetailView로 이동 / 또는 인라인으로 전문 펼침?

③ 단독 회독 카드 — "1회독" 뱃지를 표시할지, 아니면 뱃지 없이 깔끔하게 둘지?
